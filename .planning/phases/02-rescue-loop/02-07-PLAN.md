<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local .env.local → pnpm dev | Supabase secrets stay local; never committed to git |
| Cloudflare Secrets → Workers runtime | Encrypted at rest; decrypted only inside the worker sandbox |
| Supabase Storage bucket | Private bucket; no public read access; all reads via signed URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-27 | Information Disclosure | `.env.local` committed to git | mitigate | `.env.local` is in `.gitignore`; only `.env.example` is committed (with placeholder values) |
| T-02-28 | Information Disclosure | `SUPABASE_SERVICE_ROLE_KEY` in Cloudflare | accept | Cloudflare Secrets are encrypted at rest; standard cloud secrets management; acceptable for NGO scale |
| T-02-29 | Tampering | Direct Supabase API access using anon key + guessed object path | mitigate | Bucket is private — anon key cannot read or write to private buckets without a signed URL; service-role client (server-only) issues all signed URLs |
</threat_model>
