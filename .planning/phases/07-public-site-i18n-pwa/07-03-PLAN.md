NAMESPACE CONTRACT (from Plan 07-01 — request.ts returns NESTED namespaces):
```
useTranslations("common") -> t("status.en_route"), t(`foodCategory.${cat}`), t(`partnerType.${type}`), t("buttons.save")
useTranslations("portal") -> t("pickup.form.category"), t("pickup.card.claimButton"), t("notifications.empty")
useTranslations("admin")  -> t("pickups.filters.status"), t("users.actions.changeRole"), t("partners.form.name"), t("reports.metrics.servings"), t("dashboard.pickupsLink")
```
Pick the namespace per component, then key with the dotted path INSIDE that namespace (drop the namespace
prefix from the key). NEVER pass a cross-namespace key like t("admin.pickups.x") to useTranslations("admin").

Server component retrofit (apply to all async page/component files):
```typescript
// ADD import:
import { getTranslations } from "next-intl/server";
// In the async function body ADD:
const t = await getTranslations("admin"); // or "portal" / "common"
// Replace: "Apply filters" -> t("pickups.filters.applyButton")
```

Client component retrofit (apply to all 'use client' components):
```typescript
'use client';
// ADD import:
import { useTranslations } from "next-intl";
// In the component body ADD:
const t = useTranslations("admin"); // or "portal" / "common"
```

NEVER mix getTranslations (server) and useTranslations (client) in one file.
Rule: file has 'use client' directive -> useTranslations. Otherwise -> getTranslations.

Status label migration:
```typescript
// BEFORE: import { PICKUP_STATUS_LABELS } from "@/config/constants"; PICKUP_STATUS_LABELS[status]
// AFTER (client): const t = useTranslations("common"); t(`status.${status}`)
// AFTER (server): const t = await getTranslations("common"); t(`status.${status}`)
```

Food category / partner type migration:
```typescript
// FOOD_CATEGORY_LABELS[category] -> t(`foodCategory.${category}`)  (namespace "common")
// PARTNER_TYPE_LABELS[type]      -> t(`partnerType.${type}`)       (namespace "common")
```
