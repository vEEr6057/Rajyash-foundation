Server component retrofit (apply to all async page/component files):
```typescript
// BEFORE (hardcoded):
import { getTranslations } from "next-intl/server"; // ADD this import
// In the async function body: ADD
const t = await getTranslations("portal");          // or "common"
// Replace: "Claim pickup" → t("pickup.card.claimButton")
```

Client component retrofit (apply to all 'use client' components):
```typescript
'use client';
import { useTranslations } from "next-intl"; // ADD this import
// In the component function body: ADD
const t = useTranslations("portal");         // or "common"
// Replace hardcoded strings with t("...") calls
```

NEVER mix getTranslations (server) and useTranslations (client) — each component uses only one.
Key: if the file has 'use client' directive → useTranslations. Otherwise → getTranslations.

Status label migration:
```typescript
// BEFORE:
import { PICKUP_STATUS_LABELS } from "@/config/constants";
PICKUP_STATUS_LABELS[status]  // "En route"

// AFTER (server component):
const t = await getTranslations("common");
t(`status.${status}`)  // reads from common.json status.en_route = "En route"

// AFTER (client component):
const t = useTranslations("common");
t(`status.${status}`)
```

Food category migration:
```typescript
// BEFORE:
FOOD_CATEGORY_LABELS[category]
// AFTER:
t(`foodCategory.${category}`)  // reads from common.json foodCategory.cooked_meal
```
