The CSV Route Handler to implement (RESEARCH Pattern 6 — native Response, NO Buffer, NO
`export const runtime = "edge"`; opennext = Node.js runtime):
```typescript
// src/app/admin/pickups/export/route.ts
import { requireRole, AuthError } from "@/server/auth/session";
import { pickupsRepo } from "@/server/db/repositories/pickups";
import { toCsv } from "@/features/admin/lib/csv";
import { parseAdminFilters } from "@/features/admin/validations/filters";
import { FOOD_CATEGORY_LABELS } from "@/config/constants";

export async function GET(req: Request) {
  try { await requireRole(["admin"]); }            // AUTH-05 re-check (Pitfall P4)
  catch (e) {
    if (e instanceof AuthError) return new Response("Forbidden", { status: 403 });
    throw e;
  }
  const filters = parseAdminFilters(new URL(req.url).searchParams);
  const rows = await pickupsRepo.listForAdmin(filters);
  const csv = toCsv(
    ["id","status","category","quantity","unit","donorId","volunteerId","createdAt","deliveredAt"],
    rows.map(r => [r.id, r.status, r.category, r.quantity, r.quantityUnit, r.donorId,
                   r.volunteerId ?? "", r.createdAt.toISOString(), r.deliveredAt?.toISOString() ?? ""]),
  );
  return new Response(csv, {
    headers: {
      "Content-Type": "text/csv; charset=utf-8",
      "Content-Disposition": `attachment; filename="pickups-${new Date().toISOString().slice(0,10)}.csv"`,
    },
  });
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> GET /admin/pickups/export | A Route Handler under /admin — MUST re-check admin itself (middleware is not the boundary; CVE-2025-29927) |
| CSV body -> spreadsheet | Cells go through the hardened `toCsv` (formula-injection guard from 06-01) before reaching the file |
| browser -> /admin/partners, /admin/reports | Server components re-check `requireRole(["admin"])`; non-admins redirected |
| partner form input -> DB | `partnerSchema` (client + server action) gates the write; PII contact fields stored only after validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-18 | Information Disclosure | Non-admin downloads all-pickups CSV via the export URL | mitigate | The Route Handler's FIRST statement is `requireRole(["admin"])`; AuthError -> 403 (Pitfall P4) |
| T-06-19 | Tampering / RCE-on-client | A donor/volunteer/category value in the CSV executes as a formula on open | mitigate | Body built via `toCsv`/`csvCell` (06-01) which neutralizes `= + - @ tab CR` before RFC-4180 quoting; unit-tested |
| T-06-20 | Information Disclosure | Reports page over-exposing per-row data to the client | mitigate | The page ships only the 3 aggregate numbers (servings/kg/count), not rows; the aggregate runs server-side |
| T-06-21 | Input Tampering | Malformed partner create/edit payload | mitigate | `partnerSchema` validation on both the client form and the server action (06-03) |
| T-06-22 | Tampering | `setUserPartner` linking a non-donor / nonexistent user | mitigate | The action re-checks admin; the picker is sourced from `listAll` donors; setPartner only writes the partnerId column (FK enforced by the DB) |
</threat_model>
