Principles

Five rules

Feedback, not decoration

Every animation answers what happened or what's next. If it doesn't, cut it.

Transform & opacity only

GPU-composited. Never animate width, top, box-shadow or filter — they trigger layout & hurt INP.

Two budgets

App is frugal (≤200ms, functional). Public is generous (reveals, springs, count-ups).

Reduced motion is a state

Moves become fades, loops stop, content always lands. Never gate info behind motion.

Touch, not hover

Drivers tap one-handed. Design :active states; reserve hover for desktop/public.

Navigation

Page transitions

In-app: a cheap CSS enter, no exit (exits cost layout on low-end). Public: the View Transitions API for crossfades and shared-element morphs.

In-app · cheap

200ms translateY(6px)+fade, enter only. Zero JS — pure CSS class on template.tsx.

app/template.tsx
export default function Template({ children }) {
  return <div className="rj-route-shell">
    {children}
  </div>;
}
/* .rj-route-shell {
   animation: rj-route-in 200ms
     var(--ease-standard) both } */

Public · View Transitions

Crossfade + rise on route change; name shared elements to morph the hero into the detail page.

Extras & guardrails

Tasteful additions — and what to avoid

Public
Hero gradient drift
Slow 12s background-position shimmer on the CTA band. ⚠ not GPU-composited — keep the element small, never full-bleed.
Public
Image parallax on scroll
Motion useScroll → ≤24px translate. ⚠ scroll-linked → disable below 768px.
App
Status timeline fill
Progress bar via transform: scaleX(), 200ms. ✓ cheap & composited.
App
New-pickup badge bump
rj-badge-bump scale 1→1.22→1, 180ms when a count increments. ✓ cheap.
Avoid
Animating layout properties
Never animate width / height / top / left / box-shadow / filter / blur — they trigger layout & paint and wreck INP/CLS on the drivers' phones. Use transform/opacity proxies.
